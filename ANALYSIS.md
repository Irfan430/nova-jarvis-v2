# Nova Jarvis v2 — সমস্যা বিশ্লেষণ ও নতুন আর্কিটেকচার

## চিহ্নিত সমস্যাসমূহ

### ১. পরিচয় ও ব্যক্তিত্বের অভাব
- `systemPrompt` মাত্র একটি লাইন — কোনো গভীর পরিচয় নেই
- Nova Jarvis নিজেকে চেনে না, কী করতে পারে জানে না
- কোনো আবেগ, টোন, বা ব্যক্তিত্ব নেই

### ২. মেমোরি সিস্টেম নেই
- প্রতিটি `run()` কল নতুন — আগের কথোপকথন মনে নেই
- ব্যবহারকারীর নাম, পছন্দ, আগের কাজ মনে রাখে না

### ৩. টুল রেজিস্ট্রি দুর্বল
- শুধু `bash` এবং `browser_open` — মাত্র ২টি টুল
- MCP টুলগুলো stub মাত্র, আসলে কাজ করে না
- কোনো ফাইল অপারেশন, ওয়েব সার্চ, সিস্টেম কন্ট্রোল নেই

### ৪. কোনো স্ট্রিমিং নেই
- সম্পূর্ণ রেসপন্স আসার আগে কিছু দেখায় না
- রিয়েল-টাইম অনুভূতি নেই

### ৫. কোনো ইন্টারেক্টিভ CLI নেই
- শুধু test-agent.ts — কোনো লাইভ চ্যাট ইন্টারফেস নেই

### ৬. এরর হ্যান্ডলিং দুর্বল
- টুল ফেইল হলে এজেন্ট থেমে যায়

## নতুন আর্কিটেকচার

```
nova-jarvis-v2/
├── src/
│   ├── agent/
│   │   ├── Agent.ts          ← উন্নত লুপ + স্ট্রিমিং
│   │   ├── Brain.ts          ← Jarvis ব্যক্তিত্ব + সিস্টেম প্রম্পট
│   │   └── Memory.ts         ← কথোপকথন মেমোরি + ফাইল পার্সিস্টেন্স
│   ├── tools/
│   │   ├── ToolRegistry.ts   ← উন্নত রেজিস্ট্রি
│   │   ├── BashTool.ts       ← শেল কমান্ড
│   │   ├── FileTool.ts       ← ফাইল পড়া/লেখা
│   │   ├── WebTool.ts        ← ওয়েব ফেচ + সার্চ
│   │   ├── SystemTool.ts     ← সিস্টেম ইনফো
│   │   └── BrowserTool.ts    ← ব্রাউজার কন্ট্রোল
│   ├── utils/
│   │   ├── Logger.ts         ← রঙিন লগার
│   │   └── Config.ts         ← কনফিগ ম্যানেজার
│   └── cli.ts                ← ইন্টারেক্টিভ CLI
```
